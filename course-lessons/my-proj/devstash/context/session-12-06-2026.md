## Issues Resolved

### 1. Initial 404 everywhere
**Cause:** Vercel proxy was stripping `/devstash` before forwarding.
**Fix:** Changed proxy destination to `https://devstash-mu-five.vercel.app/devstash/:path*`

### 2. Bare `/devstash` gave 404
**Cause:** The wildcard rule `/devstash/:path*` requires at least one segment.
**Fix:** Added explicit rule `{ "source": "/devstash", "destination": "...vercel.app/devstash" }`

### 3. Sign-in redirected to `/api/auth/error` (404)
**Cause:** NextAuth v5 does not read `basePath` from `next.config.ts`. Without its own `basePath` option it constructs all redirect URLs without the `/devstash` prefix.
**Fix:** Added `basePath: '/devstash/api/auth'` to both `auth.ts` and `auth.config.ts`.

### 4. `UnknownAction` — Cannot parse action at `/api/auth/session`
**Cause:** Next.js strips `basePath` from URLs before route handlers see them. Handler receives `/api/auth/session` but NextAuth (with `basePath: '/devstash/api/auth'`) expected `/devstash/api/auth/session`.
**Fix:** Route handler patches the request URL to re-add `/devstash` before passing to NextAuth:
```ts
function withBasePath(req: NextRequest) {
  const url = new URL(req.url)
  url.pathname = '/devstash' + url.pathname
  return new NextRequest(url, req)
}
export function GET(req: NextRequest) { return handlers.GET(withBasePath(req)) }
export function POST(req: NextRequest) { return handlers.POST(withBasePath(req)) }
```

### 5. `GET /api/auth/providers 404` — client calls wrong auth endpoint
**Cause:** `next-auth/react` defaults to `/api/auth` basePath when no `SessionProvider` is present. With Next.js `basePath: '/devstash'`, the actual endpoint is at `/devstash/api/auth/...`.
**Fix:** Added `SessionProvider basePath="/devstash/api/auth"` via `src/components/providers.tsx` wrapped in root layout.

### 6. `env-url-basepath-mismatch` warning
**Cause:** `AUTH_URL` included the `/devstash` path, which didn't match `basePath: '/devstash/api/auth'`.
**Fix:** `AUTH_URL` set to the site root only. The warning fires when `AUTH_URL` has a non-root path that doesn't equal the configured `basePath`.

### 7. Sign-out / GitHub sign-in redirected to wrong URL
**Cause:** `next-auth/react`'s `signOut()` and `signIn()` do NOT auto-apply Next.js basePath to `callbackUrl`.
**Fix:** Hardcoded `/devstash/...` prefix in all 4 call sites (Sidebar, DeleteAccountButton, sign-in, register).

### 8. GitHub OAuth users redirected to `/verify-email`
**Cause:** Dashboard layout's `emailVerified` guard applies to all users. The Prisma adapter sets `emailVerified` on new OAuth user creation but not when linking GitHub to an existing unverified email/password account.
**Fix:** Added `signIn` callback in `auth.ts` that sets `emailVerified: new Date()` for any OAuth sign-in where it isn't already set.

## Key Architecture Notes

- **Next.js `basePath`** strips `/devstash` from URLs before BOTH route handlers AND middleware see them. `req.nextUrl.pathname` in middleware is without the prefix — path checks and `config.matcher` patterns do not need `/devstash` prefixes.
- **NextAuth `basePath`** must be `/devstash/api/auth` (for correct OAuth callback URL construction), but the route handler must re-add `/devstash` to the URL before NextAuth parses it (because Next.js has already stripped it).
- **`next-auth/react` client** (`signIn`, `signOut`, `useSession`) does NOT auto-apply Next.js basePath. Requires `SessionProvider basePath="/devstash/api/auth"` and explicit `/devstash/...` prefix in all `callbackUrl` values.
- **Next.js `redirect()`, `router.push()`, `Link href`** DO auto-apply basePath — these need no prefix.
- **Static assets** (`_next/static/...`) are served at `/devstash/_next/static/...` — the Vercel proxy's `/devstash/:path*` rule covers these automatically.
- **`AUTH_SECRET`** does not need changing — it's a signing key, not URL-dependent.
